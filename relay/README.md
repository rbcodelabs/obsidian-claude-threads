# Claude Threads Relay

A Cloudflare Worker with Durable Objects that acts as a stateless WebSocket relay between Obsidian Desktop and Obsidian Mobile instances.

## Architecture

The relay is a pure pipe. It routes frames between a desktop and a mobile client that share a room ID. It stores nothing — all thread state lives in the plugin.

Each room is a Durable Object (`RoomDO`) with two WebSocket slots:
- `role=desktop` — the desktop Obsidian instance running the Claude CLI
- `role=mobile` — the mobile Obsidian instance (read/write UI only)

## Deployment

### Prerequisites

1. A Cloudflare account with Workers paid plan (required for Durable Objects)
2. `wrangler` CLI authenticated: `npx wrangler login`

### Deploy

```bash
cd relay
npm install
npm run deploy
```

The Worker is deployed to:

```
wss://claude-threads-relay.rbcodelabs.workers.dev
```

This URL is already set as the default `relayUrl` in the plugin's `DEFAULT_SETTINGS`.

### URL Pattern

```
wss://claude-threads-relay.rbcodelabs.workers.dev/room/:roomId?role=desktop|mobile
```

- `roomId` — 32-char hex string generated by the plugin on first enable
- `role` — must be `desktop` or `mobile`

### Connection Rules

- Only one desktop and one mobile per room at a time
- A new desktop connection replaces the existing one; mobile receives a `desktop_reconnected` frame
- A second mobile connection while one is active is rejected with HTTP 409
- The relay does not implement ping/pong — the plugin handles keepalive

### Frame forwarding

All frames are forwarded verbatim between desktop and mobile. The relay does not parse or validate JSON content.

## Local development

```bash
npm run dev
```

This starts a local Miniflare server on port 8787. Useful for integration testing.
